Площадка платных редиректов на интернет-магазины
    Реализовать сервис по предоставлению площадок, которые будут иметь наборы ссылок на определенные интернет ресурсы.
    Целью сервиса, является процес платного перехода по указанным ссылкам, ведения статистики по переходам и получению коллбека по действию на конечном ресурсе.


Базовые требования к структурам данных:
	Модель Shop:
	name - название магазина;
	link - ссылка;
	price - стоимость действия;
	active - действует ли шоп;

		Модель Market:
	name - название площадки;
	alias - уникальный алиас для площадки;
	shops - Связь многое к многому -> Shop;
	active - действует ли маркет;


Базовые требования к ролям:
	Admin - имеет право создавать, удалять, изменять записи Market, Shop;
	MarketManager - Имеет право создавать, удалять, изменять Shop в предоставленных ему Market;

Базовые требования к страницам:
    Оформление:
	Все страницы должны иметь минимальное оформление при использовании html/cssjs, желательно использовать js сторонние (не самописные) библиотеки/фреймворки;

Административные:
	Admin - После авторизации может создавать юзера с ролью MarketManager и назначать ему Markets;
	Admin/MarketManager - после авторизации, имеют возможность выполнять создание, изменение, удаление записей, согласно предоставленным правам;

	Admin/MarketManager - могут просматривать следующую статистику:
		Маркет, общее количество переходов на шопы данного маркета, общее количество коллбеков от шопов данного маркера, общая стоимость переходов на шопы данного маркера;

		Под каждым маркетом список его шопов, количество переходов на каждый шоп, количество коллбеков по шопу, стоимость всех переходов по шопу;
		Неактивные шопы не учитываются в статистике по маркету и не показываются в статистике.

		Неактивные маркеты не показываются на странице статистики, все шопы, даже если они активны, никак не учитываются в неактивном маркете.

Пользовательские:
	Страница маркета - показывает все активные магазины, каждый магазин представлен названием и ссылкой для перехода, перейти на неактивный маркет нельзя, выводим оповещение, что данный маркет недоступен;

	Страница магазина имитирует магазин с одной единственной кнопкой, которая возвращает коллбек нашему сервису, а-ля пользователь купил что-то на данном магазине;

Требования к процессу:
	Предоставлять ссылки для магазинов таким образом, чтобы мы могли логгировать их переходы и хранить статистику количества переходов по каждому шопу;
	Реализовать коллбек, который будет возвращаться по нажатию кнопки на имитационной странице магазина;
	Хранить статистику количества возвращенных коллбеков для каждого шопа;
